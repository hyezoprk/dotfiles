# 작업 규칙

- 특정 GitHub 이슈를 다루는 작업(TDD, grill, 프로토타입, 버그 픽스 등)을 시작하면 즉시 tmux 윈도우 이름에 이슈 번호를 남긴다. 세션 중간에 다루는 이슈가 바뀌면 그때도 갱신한다.
  - 반드시 자기 팬을 명시 타깃으로 지정할 것: `tmux rename-window -t "$TMUX_PANE" "#369"`
  - 타깃 없이 실행하면 사용자가 보고 있는 다른 윈도우에 이름이 붙는 사고가 난다(실제 발생). `$TMUX_PANE`이 비어 있으면(tmux 밖) 조용히 건너뛴다.
  - worktree 격리로 띄운 서브에이전트(Agent 도구 등)는 별도 프로세스라 부모 세션의 `$TMUX_PANE`을 상속받지 못한다. 서브에이전트 안에서 리네임을 시도하지 말고, 부모 세션이 서브에이전트를 띄우기 전/직후에 자기 `$TMUX_PANE`으로 직접 리네임한다.
- 추적 중인 GitHub 이슈(acceptance criteria 있는)에 딸린 작업을 끝냈으면, AC 항목을 체크하고 이슈를 닫는다(`gh issue close` 등). 닫을 때 무엇을 했는지 짧게 요약한 코멘트를 남긴다.
